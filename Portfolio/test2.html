Event Aggregator - Centralized Campus Events

User Research

2025

Preview

Charan Pushpanathan Prabavathi

Overview. Information about campus events at Penn State University is scattered across multiple platforms, making discovery difficult for students. This research project focused on designing a centralized event platform with two components: Gather, a mobile app for attendees to discover events, and Post, a web tool for event organizers to create and publish events. Through user interviews and iterative design, we created a solution that addresses the fragmentation of event information and enhances the campus event experience for both attendees and organizers.

Designing a centralized platform for campus event discovery and management. UX Research, Design Methodologies, Interface. 5 event organizers and 4 event attendees at Penn State University.

🔍Problem

Information about campus events at Penn State University is scattered across multiple platforms, making discovery difficult for students and creating challenges for event organizers to reach their target audience.

✏️Research Questions

  • How do students currently discover and engage with campus events?
  • What challenges do event organizers face in promoting their events?
  • How can a centralized platform improve the event ecosystem?

🧭Approach & Method

  • 15-minute in-depth interviews with event organizers and attendees
  • Hierarchical Task Analysis to understand existing workflows
  • Synthesis & Requirements gathering
  • Iterative design and prototyping in Figma

👥Participant Profile

  • 1 event organizer from student clubs
  • 4 Penn State students who attend campus events
  • Diverse representation across academic departments

🎯Event Organizer Insights

Venue Coordination Issues

"We really get frustrated when the place changes... it feels like we spread misinformation to everybody."

Complex Event Creation Workflow

"We're a team of six members... we brainstorm ideas first, then plan logistics like budget and venue, get approvals, and finally disseminate information across multiple channels."

Limited Feedback Diversity

"Since we all know each other, they hesitate to say disappointments or complain because we are friends... So we just hear feedback from the same people every time."

Reaching Broader Audiences

"Even after we send out the posters to everybody, sometimes the place changes. It's challenging to update everyone across all the platforms we use."

👁️Event Attendee Insights

Information Fragmentation

"I get outlook mails sometimes, and sometimes my friends just call me for an event happening that's around. I get it by word." – User A

Verification Needs

"I have missed a research talk due to a lack of clear location information. I couldn't find where it was happening." – User C

Discovery Limitations

"I think a different platform or a different place where we can look at all the events, like the symplicity app I have for career fairs. Something like that." – User B

Interest Matching

"I struggle to find events that cater to my specific interests, particularly within niche areas of computer science." – User D

Social Validation

"I prioritize events my friends are attending. If they're going, I'm more likely to attend too." – User D

💡Solution

Gather Mobile App

A comprehensive mobile application designed specifically for Penn State University students to discover campus events in one centralized platform.

Post Web Platform

A specialized web platform designed for Penn State University event organizers (clubs, departments, faculty) to create, manage, and publish events directly to the Gather ecosystem.

📱Gather Features

PSU Authentication

Secure login using Penn State credentials ensures verified community access

Personalized Discovery

Interest-based event recommendations based on user preferences

Top 3 Events Showcase

Highlighting highly-rated daily events for quick discovery

Event Details Hub

Complete information including locations, times, and amenities

Tag System

Quick visual indicators for event features (refreshments, networking, speakers)

One-Click Sharing

Easy distribution of event details via WhatsApp and other platforms

RSVP Functionality

Streamlined attendance confirmation and updates

Location Guidance

Integrated maps with directions to event venues

💻Post Features

Collaborative Workspace

Team-based environment enabling multiple organizers to work together

Streamlined Event Creation

Intuitive form with structured fields for complete event information

Image Upload

Simple drag-and-drop interface for event promotional materials

AI-Assisted Content

Intelligent help with generating concise event descriptions

Location Integration

Easy venue selection with automatic mapping

Tagging System

Standardized event categorization for improved discoverability

Real-Time Updates

Changes immediately sync to the Gather app, even after publication

Penn State Authentication

Secure access limited to verified university organizers

🔄Design Process

Problem Discovery

  • Literature review on event discovery platforms
  • Observed campus announcement methods across 5+ channels
  • Identified key problems affecting Penn State event ecosystem

User Interviews

  • Conducted 15-minute in-depth interviews with 1 event organizer and 4 attendees
  • Documented pain points, current workflow, and desired solutions
  • Collected qualitative data on discovery patterns and engagement motivation

Synthesis & Requirements

  • Transformed user insights into actionable design requirements
  • Prioritized features based on user needs frequency and impact
  • Created user personas representing event organizers and attendees

Hierarchical Task Analysis

  • Mapped 10 complex tasks users perform when interacting with events
  • Analyzed steps required to complete key activities (filtering, sharing, RSVP)
  • Identified unnecessary complexity and failure points in current systems

Design & Prototyping

  • Developed low and high-fidelity prototypes in Figma
  • Followed Apple Human Interface Guidelines for consistency
  • Created two tailored solutions: Gather (mobile app) and Post (web platform)

🔮Future Work

User Testing

  • Conduct usability testing with target users
  • Refine interfaces based on feedback
  • Implement A/B testing for key features

Feature Expansion
